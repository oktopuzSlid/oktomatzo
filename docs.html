Oktomatzo Host Documentation

A full-stack platform for hosting interactive projects — 3D visualizers, AI tools, minigames, and more.

Overview

Oktomatzo Host is a registry-based project platform. Each "project" is a standalone page that lives in its own file. Adding a new project means creating a page and registering it — no routing or config changes needed.

The platform has two parts that run simultaneously:

  • Frontend (Astro + TypeScript) — serves pages, static assets, and client-side JavaScript. Runs on port 4321.
  • Backend (Python FastAPI) — handles authentication, data persistence, and heavy processing (AI, etc.). Runs on port 8000.

Setup & Run

Prerequisites

  • Node.js (v18+) — node --version
  • pnpm — npm install -g pnpm
  • Python (3.10+) — python --version

One-Command Start

pnpm run dev:all

This starts both servers. Open http://localhost:4321 in your browser.

Separate Terminals

# Terminal 1: Frontend
pnpm dev

# Terminal 2: Backend
cd backend
.venv\Scripts\activate
uvicorn app.main:app --reload --port 8000

Adding a New Project — Step by Step

Adding a project to Oktomatzo Host takes 5 steps. The system is designed so each project is fully independent — you can use any frontend technology (Three.js, Canvas, React, vanilla JS) inside it.

1 Create the Astro Page

src/pages/projects/my-new-project.astro

---
import Base from '../../layouts/Base.astro'
---

<Base title="My New Project | Oktomatzo Host">
  <section class="page-section">
    <div class="container">
      <a href="/projects/" class="back-link">&larr; Back to Projects</a>
      <h1 class="page-title">My New Project</h1>
      <p class="page-description">Description of what this project does.</p>

      <!-- Your project content here -->
      <div id="project-root" class="project-root"></div>
    </div>
  </section>
</Base>

<!-- Load your project's JavaScript -->
<script src="/js/projects/my-new-project.js" defer></script>

The <Base> layout automatically adds the header, footer, nav, and global CSS.

2 Register in the Project Registry

src/lib/projects.ts

import type { ProjectDefinition } from './types'

export const projects: ProjectDefinition[] = [
  // ... existing projects ...
  {
    slug: 'my-new-project',
    title: 'My New Project',
    description: 'What this project does in one line.',
    category: 'interactive',        // 'threejs' | 'data-viz' | 'interactive' | 'tool'
    icon: 'smile',                   // matches SVG in ProjectCard.astro
    href: '/projects/my-new-project/',
  },
]

The card appears on /projects/ automatically. No HTML changes needed.

3 Add Client-Side Logic

public/js/projects/my-new-project.js

// Plain JavaScript — no framework required
document.addEventListener('DOMContentLoaded', function () {
  const root = document.getElementById('project-root')
  if (!root) return

  root.innerHTML = '<h2>My project is running!</h2>'
})

For Three.js projects, use an import map in the .astro page and type="module" on the script tag. See 3d-viz.astro for reference.

4 (Optional) Add Backend Endpoints

backend/app/routers/projects.py

from fastapi import APIRouter, Depends
from ..auth import get_current_user

router = APIRouter(prefix="/api", tags=["projects"])

@router.post("/my-endpoint")
def my_endpoint(user_email: str = Depends(get_current_user)):
    if not user_email:
        return {"error": "Authentication required"}
    # Your logic here
    return {"result": "success"}

Add the router to backend/app/main.py: app.include_router(my_router)

5 (Optional) Add Frontend API Client Functions

src/lib/api.ts

export async function myEndpoint(data: any): Promise<any> {
  const token = localStorage.getItem('token')
  const res = await fetch(`${API_BASE}/api/my-endpoint`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      ...(token && { 'Authorization': `Bearer ${token}` }),
    },
    body: JSON.stringify(data),
  })
  if (!res.ok) throw new Error('Request failed')
  return res.json()
}

Best Practices for Adding Projects

  • Keep each project independent. Don't share mutable state between projects.
  • Use the #project-root div as the mounting point for client-side logic.
  • For Three.js: use an import map in the .astro page — no npm needed.
  • For backend calls: always check user_email is not None before processing.
  • Rate limit new endpoints with @limiter.limit("N/minute").
  • Validate all inputs with Pydantic schemas.
Every project is served from the same domain. You can mix Three.js, Canvas, Phaser, React, or vanilla JS — each page loads only its own code.

Database (SQLite)

The database file is backend/database.db and is auto-created on the first run. Two tables:

users
  id            INTEGER PRIMARY KEY AUTOINCREMENT
  name          TEXT NOT NULL
  email         TEXT UNIQUE NOT NULL
  password_hash TEXT NOT NULL           (bcrypt hashed)
  created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP

projects
  id            INTEGER PRIMARY KEY AUTOINCREMENT
  slug          TEXT UNIQUE NOT NULL
  title         TEXT NOT NULL
  data          TEXT                    (JSON or prompt text)
  user_email    TEXT FK -> users.email
  created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP

Authentication Flow

  1. Signup → password is hashed with bcrypt, stored in DB. A JWT token is returned.
  2. Login → password is verified against the hash. A JWT token is returned.
  3. Requests → the JWT token is sent in the Authorization: Bearer <token> header.
  4. Verification → get_current_user() decodes the JWT, returns the user's email.
  5. Expiry → tokens expire after 60 minutes.

API Client (src/lib/api.ts)

The API client provides typed functions for all backend endpoints. Import it in any project page:

import { checkHealth, login, signup, generateImage, isLoggedIn, logout } from '../../lib/api'

// Check if backend is alive
const health = await checkHealth()

// Sign up
const { access_token, user_name } = await signup('Alice', 'alice@example.com', 'password123')

// Log in
const tokenData = await login('alice@example.com', 'password123')

// Call generate (requires auth)
const result = await generateImage('a purple torus knot')

The token is stored in localStorage automatically by the login() and signup() functions.

Security Measures

  • Password hashing: bcrypt via passlib — passwords never stored in plaintext.
  • JWT tokens: expire after 60 minutes, signed with SECRET_KEY from .env.
  • Input validation: Pydantic schemas enforce length limits (prompt ≤ 1000 chars, password ≥ 8 chars).
  • SQL injection: All queries use parameterized placeholders (?).
  • Rate limiting: 10-30 requests per minute per endpoint via slowapi.
  • CORS: Whitelist configured in .env — only specified origins can call the API.
  • Directory traversal: File operations restricted to storage/ directory.
The SECRET_KEY in .env defaults to a placeholder. Change it to a random string before deploying to production. Generate one with: python -c "import secrets; print(secrets.token_hex(32))"

Deployment Notes

Frontend (Astro):

pnpm build     # Output goes to dist/
# Deploy dist/ to any static host: Vercel, Netlify, GitHub Pages, Cloudflare Pages

Backend (Python):

# Option A: Direct with uvicorn
cd backend
.venv\Scripts\activate
uvicorn app.main:app --host 0.0.0.0 --port 8000

# Option B: Docker
docker build -t oktomatzo-backend .
docker run -p 8000:8000 oktomatzo-backend

CORS: Update CORS_ORIGINS in backend/.env to include your deployed frontend URL.
