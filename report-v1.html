Version 1.0 Release Report

v0.3 → v1.0

Comprehensive comparison of architecture, technologies, and capabilities before and after the backend integration.

Before vs After

Before (v0.3)

Static frontend only
  • Astro + TypeScript frontend
  • Static HTML pages, no backend
  • No user authentication
  • No data persistence
  • Import map for Three.js CDN
  • All projects client-side only
  • No API endpoints
  • No rate limiting
  • No input validation
  • Single dev command (pnpm dev)

After (v1.0)

Full-stack architecture
  • Astro + TypeScript frontend
  • Python FastAPI backend
  • User signup / login with JWT
  • SQLite database persistence
  • Import map for Three.js CDN
  • Projects can call backend APIs
  • 6 REST API endpoints
  • Rate limiting on all endpoints
  • Pydantic input validation
  • Single command: pnpm run dev:all

How to Run v1.0

# Terminal 1 (one command):
pnpm run dev:all

# Or two terminals:
# Terminal 1: Frontend
pnpm dev

# Terminal 2: Backend
cd backend
.venv\Scripts\activate
uvicorn app.main:app --reload --port 8000

Database Schema

users
├── id              INTEGER PRIMARY KEY
├── name            TEXT NOT NULL
├── email           TEXT UNIQUE NOT NULL
├── password_hash   TEXT NOT NULL (bcrypt)
└── created_at      TIMESTAMP

projects
├── id              INTEGER PRIMARY KEY
├── slug            TEXT UNIQUE NOT NULL
├── title           TEXT NOT NULL
├── data            TEXT (JSON or prompt)
├── user_email      TEXT FK -> users.email
└── created_at      TIMESTAMP
